# 구현 상태 · v0.3.18

현재 로컬 MVP는 로그인·홈·조사조 구성·브리핑·조사·결과 전 화면 게임보이 UI, Y콤퓨타체를 사용하는 본문·SYSTEM·입력 UI와 제한된 포인트 강조를 포함한 7종 로컬 폰트 역할 체계, 실제 조사 노드·이동 경로와 일치하는 층별 건물 평면 안내도·현재 위치 붉은 점·요청형 팝업, 동일 변주 전역 아이템 선점과 멀티탭 동시 요청 직렬화, 활성 세션의 잘리지 않는 SYSTEM·채팅 타임라인, 이동 경로·세부 위치 단위 현장 동기화, 이미지 매핑과 플레이스홀더, 오브젝트 상세 팝업을 포함합니다. 조사 화면은 가로·세로 드래그 분할선으로 일러스트/SYSTEM/채팅 비율을 조절하고 브라우저에 저장할 수 있습니다. 현재 장소 재이동은 위치 확인으로 끝나며, 비인접 목적지는 directed route 최단 경로의 첫 방향과 이후 순서를 SYSTEM으로 안내합니다. 캐릭터 행동 원문은 SYSTEM에 보존하며, AI가 행동 주체·감각 방식·대상·실행 가능성·위험 관련성·사용 물건을 구조화합니다. 기존 결정 규칙이 이동·위험·오염·아이템 상태를 먼저 확정한 뒤 AI가 그 사실을 바꾸지 않는 몰입형 결과문을 생성하며, API 오류 시 행동별 로컬 문장으로 자동 전환합니다.

## 이전 구현 상태 · v0.3.3


## v0.3.3 채팅 범위·입력 초기화

- 전송 성공 시 입력 textarea의 DOM 값과 UI 상태를 동시에 초기화
- 일반 채팅 로그만 `interaction` 타입으로 저장
- `/행동`은 채팅 복제 없이 서버 판정 흐름으로만 전달
- 채팅 로그에 `scopeKey`를 저장해 현재 위치 또는 이동 경로별로 분리
- 현재 `scopeKey`와 일치하는 채팅만 렌더링
- 조사·획득 결과는 actor chat이 아니라 SYSTEM 로그로 출력

## v0.3.2 입력 트리거

- 일반 문장은 캐릭터 대화로만 전송하며 시스템이 해석하지 않습니다.
- `/`로 시작한 문장만 이동·조사·위험 대응·힌트 요청으로 판정합니다.
- 위험 대응 중에도 일반 채팅은 자유롭게 전송할 수 있습니다.
- 선택지 클릭 시 입력창에는 `/행동` 형태로 들어갑니다.

## 자유 입력 판정 계층

현재 로컬 MVP는 다음 순서로 입력을 해석합니다.

1. 복수 행동 여부 검사
2. 지도·듣기·자기 상태 확인·관찰·길 묻기처럼 상태를 소모하지 않는 의도 우선 분리
3. 현재 위치에서 직접 연결된 route 명칭·오타·구어체 표현 비교
4. 현재 장소 오브젝트·세부 지점 명칭과 비교
5. OpenAI JSON Schema로 행동 주체·감각·대상·실행 가능성·위험 관련성과 후보 ID 보완
6. 허용된 후보 ID만 통과시킨 뒤 기존 규칙으로 실제 결과 확정
7. 기존 규칙으로 상태를 확정한 뒤 AI가 사실 고정형 결과문 생성
8. API 미설정·오류 시 확장된 로컬 판정·서술로 자동 전환

AI가 성공·실패나 수치를 직접 바꾸는 구조는 아닙니다. 로컬 규칙이 최종 판정하고 AI 결과문에는 확정된 사건만 전달합니다. 향후 서버 전환 시에도 최종 판정은 RPC가 담당해야 합니다.
