## 이번 작업의 목표

이번에는 전체 10일 시스템을 한 번에 만들지 않습니다.

**해오름역 1일차를 대상으로 로그인 → 조사조 구성 및 확정 → 전원 준비 → 조사 시작 → 비공개 시간 변주 배정 → 실제 인접 경로 이동 → 장소/디테일/오브젝트 조사 → 별도 아이템 획득 → 복수 위험 순차 해결 → 서버 오염 판정 → 실시간 조원 동기화 → 재접속 복구까지 가능한 MVP 수직 단면을 구축합니다.**

## 절대 규칙

1. 원본 문서와 XLSX를 먼저 읽으세요.
2. XLSX의 시트명·헤더·ID를 직접 확인하세요.
3. 원본에 없는 세계관 콘텐츠, 단서, 장소, 아이템, 정답을 만들지 마세요.
4. 시간 변주 코드 `a/b/c/d`, 내부 위험도, seed, condition ID를 유저 화면에 노출하지 마세요.
5. 이동은 directed route 한 edge씩 처리하고 텔레포트를 금지하세요.
6. 같은 층이어도 직접 연결되지 않은 장소로 이동할 수 없습니다.
7. 조사와 아이템 획득은 반드시 별도 행동입니다.
8. 복수 위험은 한 채팅에 한 위험만 해결합니다.
9. AI는 행동 해석과 서술만 담당하고 성공·실패·오염·상태 변경은 서버가 확정합니다.
10. 오염 수치는 XLSX `26_오염계산`만 사용합니다.
11. 모든 mutation은 Supabase RPC로 처리하고 클라이언트가 DB 상태를 직접 쓰지 않게 하세요.
12. RLS를 모든 공개 테이블에 적용하고 다른 조사조의 비공개 데이터가 보이지 않게 하세요.
13. AI 실패 시 수동 판정을 요구하지 말고 서버 fallback text를 즉시 표시하세요.
14. 기능을 한 번에 전부 작성하지 말고 작은 단계로 구현·테스트하세요.

## MVP 화면

### `/login`

- 폐쇄형 로그인
- 공개 회원가입 없음
- 로그인 오류와 운영 공지

### `/onboarding`

- 연결된 캐릭터 확인
- 다른 캐릭터 선택 없음

### `/home`

- 스토리 일차
- 현실 시각
- 내 상태·오염도
- 인벤토리 요약
- 받은 초대
- 진행 중 세션 복귀
- 조사조 만들기

### `/party/new`

- 현재 허용된 목적지만 표시
- story day 1은 해오름역만
- 조원 초대
- 1인 조사 허용

- 참가자
- 초대 상태

- 준비 완료

- 공개 가능한 목적지와 환경
- 조작법

- 상단: 일차·현실 시각·구역·현재 위치·연결 상태
- 왼쪽: 조원과 접속 상태

- 오른쪽: 이동/주변/인벤토리/상태/기록
- 하단: 자유 행동 입력과 서버 제공 action label

## 첫 완료 시나리오

두 브라우저에서 두 유저가 로그인하고, 같은 조사조를 구성한 뒤 전원 ready로 세션을 시작합니다. 한 유저가 이동을 선택하면 두 화면에 같은 위험 장면이 나타납니다. 위험 두 개를 서로 다른 두 행동으로 해결한 뒤 도착하고, 오브젝트를 조사한 다음 별도 가져가기 행동으로 아이템을 획득합니다. 한 유저가 연결을 끊었다가 돌아와도 현재 위치와 장면이 복구되어야 합니다.

이 시나리오가 통과하기 전에는 게시판, 전화, A~D, 1001편, ESCAPE_ONLY, 공동 목표, 엔딩을 구현하지 마세요.

- 후보 이외의 자유 행동도 AI가 의도를 구조화하고 서버 RPC가 판정
- 색상 추가 금지, 선화와 하프톤 기반 흑백 유지
