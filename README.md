# 백지도시 조사 홈페이지 MVP v0.3.18

해오름역 1일차를 대상으로 만든 **실행 가능한 로컬 프로토타입**입니다.

## 실행 주소

```text
http://localhost:4173
```

## v0.3.18 조사 화면 크기 조절·목적지 경로 안내

- 일러스트와 SYSTEM 로그 사이 가로 조절선을 드래그해 두 창의 높이를 바꿀 수 있습니다.
- 왼쪽 일러스트·로그 영역과 오른쪽 채팅 영역 사이 세로 조절선을 드래그해 너비를 바꿀 수 있습니다.
- 조절한 비율은 브라우저에 저장되며, 방향키 미세 조절과 더블클릭 초기화도 지원합니다.
- 채팅 탭 아래의 중복된 `조사 채팅 · 1인 조사조 · ●1` 요약창을 제거했습니다.
- 현재 장소를 목적지로 다시 입력하면 선택지 대신 `이미 …에 있다`는 SYSTEM 문장을 출력합니다.
- 직접 연결되지 않은 목적지는 다른 선택지를 열지 않고 최단 경로의 첫 이동 방향과 이후 연결 순서를 안내합니다.
- 모바일·좁은 화면에서는 기존 세로 배치를 유지하며 조절선은 자동으로 숨깁니다.
- 기존 저장 키, AI 행동 분석, 전체 로그 보존, 조사·채팅·이동·위험·아이템 기능은 그대로 유지합니다.

## v0.3.17 AI 행동 분석·로그 영구 보존·자연어 결과 개선

- 활성 조사 세션의 SYSTEM·조사 채팅 기록 개수 제한을 제거했습니다.

## v0.3.15 행동 원문 기록·사용 물건 선행 판정

- `/행동`으로 입력한 원문을 SYSTEM 로그에 `[캐릭터 아이콘] 캐릭터명: (행동 지문)` 형태로 먼저 보존합니다.
- 행동 문장에서 꺼내거나 사용하는 물건 이름을 로컬 규칙과 AI가 함께 식별합니다.
- AI에는 현재 캐릭터가 실제로 가진 소지품의 이름·수량·상태·용도 태그만 전달합니다.
- 언급한 물건이 없으면 준비 단계에서 실패하고 이동·위험 진행·오염 계산을 실행하지 않습니다.
- 물건을 가지고 있어도 파손·소진·오염 상태이거나 용도와 맞지 않으면 구체적인 이유를 출력하고 후속 행동을 중단합니다.
- 사용할 수 있는 물건이면 꺼내고 사용하는 준비 동작을 이동 또는 위험 대응 결과 앞에 자연스럽게 이어서 출력합니다.
- 기존 저장 키와 조사·채팅·이동·위험·행동 선택·아이템 선점 기능은 그대로 유지합니다.

- 화면 전체를 덮던 점무늬 오버레이를 제거해 카드·세션 정보·조사 상황 문장 위에 패턴이 겹치지 않게 했습니다.
- 로그인 소개, 홈·조사조·결과의 상단 설명 영역을 브리핑처럼 무늬 없는 단색 배경으로 변경했습니다.
- 구역 지도에서 글자 뒤 격자·사선 패턴과 층을 길게 가로지르던 점선을 제거했습니다.
- 지상은 `구역 입구 → 환승광장 → 동·서 출입구`, B1은 `중앙 대합실 → 발매 구역·개찰구·제4쉘터`, B2는 `환승 통로 → 승강장` 구조가 바로 보이도록 재배치했습니다.
- 층간 이동은 긴 선 대신 각 계단 위치에 연결 층과 목적지를 직접 표기했습니다.
- 실제 조사 노드 11개, 왕복 이동 경로 22개, 현재 위치 붉은 점과 기존 지도 팝업 기능은 그대로 유지했습니다.

- `run.bat` 실행 시 Node.js 버전과 고정 실행 주소를 먼저 표시합니다.

## v0.3.12 AI 자유문장 행동 판정

- `/지도 봄`, `/이동한당`, `/동부 출입구로 감`, `/계단을 따라간다`, `/개찰구로 가자`처럼 맞춤법·어미·문장 구조가 달라도 행동 의도와 실제 후보 대상을 해석합니다.
- 명확한 지도·관찰·오브젝트·목적지 표현은 즉시 로컬 판정하고, 규칙만으로 확정하기 어려운 자유문장은 OpenAI API로 보완 판정합니다.
- AI는 의도와 대상만 고릅니다. 이동 가능 여부, 위험 성공·실패, 오염 변화, 아이템 획득은 기존 게임 규칙이 계속 확정합니다.
- 위험 중 지도·관찰·길 묻기는 위험 대응으로 소모하지 않으며, 물리적인 회피·통과 행동만 위험 대응으로 처리합니다.
- API가 꺼져 있거나 일시적으로 응답하지 않아도 확장된 로컬 판정으로 자동 전환합니다.
- 브라우저에는 API 키를 전달하지 않습니다. `server.mjs`가 운영체제 환경 변수의 키를 읽어 OpenAI API와 통신합니다.
- 입력창 아래에서 `AI 자유문장 판정` 또는 `로컬 판정 대기` 상태를 확인할 수 있습니다.
- 기존 저장 키와 조사·채팅·이동·위험·행동 선택 기능을 유지합니다.

## v0.3.11 위험 의도 판정·건물 안내도·변주별 아이템 선점

- 위험 상황에서도 지도·길찾기·관찰 의도를 위험 대응보다 먼저 구분합니다.
- `/지도 봄`, `/지도 보기`, `/지도 봤음` 같은 명사형·축약형 표현도 지도 요청으로 처리하며 위험을 해결하거나 소모하지 않습니다.
- 지도는 노드 상자 구조도에서 층별 외벽·방·복도·출입문·계단실이 보이는 건물 평면 안내도로 다시 제작했습니다.
- 현재 위치의 구역에는 붉은 점을 표시하고, 팝업 상단에도 붉은 위치 표식과 장소명을 함께 표시합니다.
- 아이템 획득 여부를 세션별이 아니라 동일 변주 전체에서 공유합니다.
- 동일 변주의 다른 캐릭터가 가져간 물품은 행동 선택지에서 제거됩니다.
- 갱신 전 선택지를 간발의 차로 눌렀다면 `이미 누군가 가져가버렸다···` 획득 실패 SYSTEM 문장을 출력합니다.
- 획득 물품이 없는 오브젝트를 조사하면 `내부를 확인해도 아무것도 없다.` 문장을 출력합니다.
- 기존 저장 키와 조사·채팅·이동·위험·행동 선택 기능은 유지합니다.

- 일러스트 우측 상단의 용도를 알 수 없던 `!` 박스를 제거했습니다. 현재 위험 설명은 좌측 하단 SYSTEM 창에만 표시됩니다.
- 행동 선택지는 더 이상 우측 채팅 로그를 밀어내지 않습니다. 일러스트 우측 하단의 오버레이로 표시됩니다.
- 행동 선택 오버레이에는 닫기 버튼을 제공하며, 닫은 뒤에는 작은 `행동 선택` 버튼으로 다시 열 수 있습니다.
- 새 선택지가 열릴 때 0.15초의 짧은 단계식 슬라이드 애니메이션을 적용했습니다.
- 세부 조사 지점을 선택하면 실제 현장 범위에도 반영되어 같은 세부 위치의 캐릭터만 현장 인원과 대화를 공유합니다.


## v0.3.4 로그인·채팅 타임라인·행동 판정 개선

- 사이트 첫 접속 시 캐릭터 선택 카드 대신 아이디·비밀번호 로그인 창이 표시됩니다.
- 테스트 계정은 `캐릭터A`, `캐릭터B`, `캐릭터C`이며 비밀번호는 모두 `1234`입니다.
- 이전 장소에서 나눈 대화를 숨기지 않고 조사 채팅에 계속 누적합니다.
- 장소 또는 이동 경로가 바뀔 때 채팅창에 흑백 구분선을 넣어 대화가 오간 위치를 구분합니다.
- 일반 대화는 행동 판정을 발생시키지 않습니다. 다만 `어디로 가야 하지?`, `길을 잃었어`처럼 명백히 길을 묻는 대화는 힌트 요청으로만 해석해 선택지를 엽니다.
- `/관찰한다`는 이동이나 위험 해결로 처리하지 않고, 현재 위치 또는 이동 경로에서 육안으로 확인되는 내용만 출력합니다.
- `/지나간다`, `/이동한다` 등은 현재 경로 수와 위험 상태를 구분해 판정합니다. 목적지가 여러 곳이면 임의 이동하지 않고 갈림길을 묘사한 뒤 선택지를 엽니다.
- 위험 대응 결과를 입력 문장의 어미에 기계적으로 이어 붙이지 않고, 행동 의도·위험 종류·성공 정도에 맞는 장면형 문장으로 다시 작성합니다.
- `현재 장면`, `시도`, `대상 지정`, `요청 거부`, `서버 스냅샷`처럼 몰입을 깨는 표현을 조사 중 시스템 문장에서 제거했습니다.


## v0.3.3 채팅 범위와 입력 처리 수정

- 일반 채팅과 `/행동` 전송 후 입력창을 즉시 비웁니다.
- `/행동` 원문은 조사 채팅에 표시하지 않고 판정 결과만 SYSTEM 창에 기록합니다.
- 조사·획득 버튼으로 실행한 행동 결과도 SYSTEM 창에 출력합니다.
- 일반 채팅에는 메시지를 보낸 위치 범위를 저장합니다.
- 채팅은 발화 당시 같은 위치에 있던 인물에게만 전달됩니다.
- 이후 다른 장소로 이동해도 이미 들은 대화는 채팅 타임라인에 남습니다.
- 장소가 달라질 때 위치 구분선이 삽입되며, 당시 다른 장소에 있던 조사조의 대화는 전달되지 않습니다.

## v0.3.2 일반 채팅과 시스템 행동 분리

- 슬래시 없이 입력한 문장은 같은 조사조와 같은 현장의 캐릭터에게 전달되는 일반 채팅입니다.
- 일반 채팅에는 이동·조사·위험 대응·오염·선택지 공개 등 어떤 시스템 판정도 발생하지 않습니다.
- 시스템 판정이 필요한 행동은 문장 맨 앞에 `/`를 붙여 입력합니다.
  - 예: `/잔상 주기를 확인한다`
  - 예: `/동부 출입구로 이동한다`
  - 예: `/안내판을 조사한다`
  - 예: `/어떻게 대응해야 하지?`
- `/행동` 원문은 채팅창에 표시하지 않으며, 시스템 판정 결과만 좌측 SYSTEM 창에 출력됩니다.
- 행동 선택지 버튼을 누르면 입력창에 `/행동` 형태로 들어갑니다.
- 위험이 발생한 상태에서도 슬래시 없는 일반 대화는 자유롭게 주고받을 수 있습니다.

## 바로 실행하기

### Windows

`run.bat`을 더블클릭합니다.

또는 프로젝트 폴더에서:

```bat
npm run dev
```

브라우저에서 `http://localhost:4173`으로 접속합니다.

`setx OPENAI_API_KEY "발급받은_키"`로 한 번 저장했다면 업데이트 때마다 다시 입력할 필요가 없습니다. 서버는 현재 프로세스에 키가 없을 때 Windows 사용자 환경 변수에서 키를 읽으며, 키 값 자체는 화면에 출력하지 않습니다.

창에는 `Node version`과 `http://localhost:4173`이 표시되어야 합니다. 오류가 생기면 창이 자동으로 사라지지 않고 종료 코드와 해결 안내를 표시합니다.

### macOS / Linux

```bash
chmod +x run.sh
./run.sh
```

브라우저에서 `http://localhost:4173`으로 접속합니다.

현재 터미널에 `OPENAI_API_KEY`가 설정되어 있어야 AI 판정이 활성화됩니다. 키가 없으면 사이트는 실행되며 로컬 판정으로 자동 전환됩니다.

> v0.3.12부터 AI 판정 API가 포함되어 있으므로 `python -m http.server` 대신 `run.bat`, `run.sh` 또는 `npm run dev`로 실행해야 합니다. Node.js 18 이상이 필요하며 외부 npm 패키지 설치는 필요하지 않습니다.

## 여러 조사조가 만나는 기능 테스트

1. 같은 브라우저에서 탭을 세 개 엽니다.
2. 첫 탭은 `캐릭터A`, 두 번째 탭은 `캐릭터B`, 세 번째 탭은 `캐릭터C`로 로그인합니다. 비밀번호는 모두 `1234`입니다.
3. A는 1인 조사조를 구성합니다.
4. B는 별도 조사조를 만들고 C를 초대합니다.
5. 각 조사조가 준비를 끝내고 세션을 시작합니다.
6. 두 조사조가 같은 장소에 도착하면 다음 알림이 나타납니다.
   - 먼저 있던 조사조: `누구가 방금 도착해 현장에 합류했다.`
   - 도착한 조사조: `먼저 도착해 있던 누구와 마주쳤다.`
7. 오른쪽 `교류` 탭에서 대화를 전송하거나 소지품을 건넵니다.
8. 한 조사조가 이동하면 남은 조사조에는 떠났다는 알림이 표시됩니다.

현재 사용자 로그인은 탭별 `sessionStorage`, 공용 조사 상태는 `localStorage`에 저장하며 브라우저의 `storage` 이벤트로 탭 사이를 동기화합니다.
